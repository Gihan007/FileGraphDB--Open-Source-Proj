# Code of Conduct

FileGraphDB should be a welcoming project for beginners and experienced
contributors alike.

## Our Standards

Examples of behavior that helps the project:

- Be respectful and patient.
- Give clear, constructive feedback.
- Welcome beginner questions.
- Focus criticism on ideas and code, not people.
- Assume good intent when discussing issues or pull requests.

Examples of behavior that is not acceptable:

- Harassment, insults, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeatedly derailing issues or pull requests.
- Any conduct that makes participation unsafe or hostile.

## Reporting Problems

If you see behavior that violates this code of conduct, contact the project
maintainer privately. Until a dedicated contact address is added, use GitHub
Issues only for non-sensitive concerns.

Maintainers may remove comments, close issues, reject contributions, or block
contributors when needed to protect the community.
